# MordenCPP_EX

## 연산자 오버로딩 정리

### 연산자 오버로딩 
	operator 키워드와 연산자를 묶어서 함수의 이름을 정의하면   
	함수의 이름을 이용한 함수의 호출 뿐만 아니라   
	연산자를 이용한 함수의 호출 가능  


### 연산자를 오버로딩 하는 두가지 
	- 멤버함수에 의한 연산자 오버로딩   
		Point operator+(const Point &ref) 
	- 전역함수에 의한 연산자 오버로딩   
		Point operator+(const Point &pos1, const Point &pos2)

### 연산자 오버로딩 구현시 주의 사항
	- 오버로딩 불가능한 연산자 존재 (예: . , .* , :: , sizeof , static_cast 등등...)
	- 멤버함수 기반으로만 오버로딩 가능한 연산자 존재 (예: = , () , [] , ->)
	- 본래의 의도를 벗어아지 않은 형태
	- 연산자의 우선순위와 결합성은 바뀌지 않음
	- 매개변수의 디폴트값 설정이 불가능
	- 연산자의 순수 기능까지 빼앗을 수는 없음

### 단항 연산자 오버로딩에서 전위증가, 후위증가 구분
	- int 키워드를 사용하여 구분
	- int 키워드는 단지 구분하기 위한 목적일 뿐 데이터 형태로써 의미는 없음	
		++pos -> pos.operator++()
		pos++ -> pos.operator++(int)
	- 전위, 후위의 차이는 증감 연산의 자체 값과 연산 결과 값의 차이를 보면 알 수 있다.
		int a = 1, b = 1;
		printf(" %d, %d \n", ++a, b++); // 연산의 자체 값 출력 결과 > 2, 1
		printf(" %d, %d \n", a, b);		// 연산 결과 값 출력 결과 > 2, 2

### 연산자 오버로딩의 교환법칙
	- 교환법칙 : A+B = B+A
	- 전역함수에 의한 연산자 오버로딩을 사용하여 구현해야함.
